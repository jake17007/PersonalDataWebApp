Developer Guide

A guide to creating apps for the Connectrix platform

Introduction

What is Connectrix?

Connectrix is a platform for secure personal data analytics. Our software aggregates the data, feeds it to the app you've created for our platform, and delivers the output to the user. In this way, the user can benefit from a community of developers/analysts like you who make apps for them, but they don't have to worry about spreading their personal data all over the web.

How does it work?

As a developer, you create a Python program that accepts preformatted JSON data from any of the third-parties you require for your app -- e.g. Fitbit. Your program accepts the data, does some calculation, formats the output to HTML/CSS/JavaScript, and sends it off for us to render to the user. To maintain the security of the user's personal data, you store your program on Github; when the user wants to use your app, we retrieve it and run it in a sandbox on our secure servers.

Before Starting

Required

Github Account

You'll need a Github account to store your program. It should be in it's own public repository.

Highly Recommended

Fitbit Example Data

In order to test your program, we recommend using your own data. This means that for any connections you will require for your app, you should have those connections established on your own Connectrix user account. For the Getting Started Tutorial, a Fitbit connection will be used. Even if you don't have a Fitbit, you should create an account on Fitbit.com. Be sure to enter your weight and height when setting up your profile; we'll be retrieving and analyzing that information via their API. If you really don't want to do this, you can use the example data provided in the test engine below.

Node.js

In order to test your program on your local machine before publishing it, we recommend installing Node.js to run the test engine provided below. You can find installation instructions at Nodejs.org.

Getting Started Tutorial

This tutorial will guide you through creating you first app: an app that prints out "Hello, ..." and your Fitbit username. As stated above, you should sign up for a Fitbit account at Fitbit.com and establish the connection on Connectrix.

Create a New App

  1. Navigate to the Create a New App page.
  2. Give your app a name. (All of this information can be changed later.)
  3. Indicate the url of the public Github repository where you'll be storing your program.
  4. Give your app a description.
  5. Select the third-party data that will be required for your app. For this tutorial, only select Fitbit > Profile.
  6. Click save.

Write Your App

Test Engine

Download/clone this sample program to your local environment to begin writing your app. It's a Node.js program that feeds data to your program, and after your program has delivered output, it's written to an HTML file.

Copy/Paste Example Data

  1. Make sure you have established the Fitbit connection for your Connectrix user account. If you haven't, you can do so on the Connect page.
  2. From the Developers page, click your new app, then click "Edit App" at the upper right of the page.
  3. Click "View JSON" at the bottom of the page.
  4. Click "Copy to Clipboard" at the upper right of the page.
  5. Paste the your data in the data.json file of the Test Engine. (Replace the existing data.)

Give It a Test Run

  1. Open your Terminal or Command Prompt.
  2. Navigate to the Test Engine directory.
  3. Run node runApp.js.
  4. Located in the Test Engine directory, open view.html file in your browser. You should see your Fitbit username.

Edit Your Program

  1. Open app.py in a text editor. This is the main script that is exectuted by Connectrix which accepts the data and returns output.
  2. Edit the line with your fitbit user name to say username = 'Hello, ' + username, and save the file.
  3. Run node runApp.js again.
  4. When you refresh your browser pointed at view.json, you should now see, "Hello, ..." by your Fitbit username.

Push to Github and Test Online

  1. Push your program to your Github repository. You may push the entire Test Engine directory or just the necessary Python files.
  2. Navigate to the Developers page, click your app, and click "Use App" at the upper right of the page.
  3. You should be redirected to you Dashboard, where your new app should be listed.
  4. Click to use the app and you're done!
